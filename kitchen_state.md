# kitchen_state.md v2

## Purpose

This document defines the stable kitchen operating state of Acquerello for scheduling purposes.

It captures:
- the core AM and PM station structure
- family meal coverage
- leadership and line roles
- each staff member’s current capabilities
- practical scheduling rules that should guide weekly staffing decisions

This file should contain only relatively durable truths.
Short-term changes such as vacations, temporary shortages, special events, or weekly training adjustments should live in a separate weekly dynamic layer.

## Practical Scheduling Rules

### 1. Expedition
- Chef is the default primary expeditor
- CDC is usually the second expediter or expedition support
- If CDC is unavailable, Raimi can cover expedite
- In slower services, one expediter is enough

### 2. Peak vs Mid-level vs Slow Service
**Peak service**
- each PM station should have dedicated coverage
- 1 floater
- 2 expeditors

**Mid-level service**
- each PM station should have dedicated coverage
- 2 expediter
- no floater 

**Slow service**
- Pasta + Mids may be combined
- Stuzz + Pastry may be combined
- expediter + pantry may be combined
- One expeditor may be used instead of two
- no floater needed

### 3. Hot Line Priority
- Sam or Kate should be prioritized for pasta
- If you don't need Raimi for expediter or floater, he then should be prioritized for hot-side coverage, first pasta, then Sauté, lastly mids
- Brandon should currently be treated as Sauté-first

### 4. Cold Side Priority
- Mateo is the default Pantry assignment
- Sebastian is the main flexible cold-side support
- Echo is currently limited to Stuzz

### 5. Pastry
- AJ is the default PM pastry person
- AM pastry team should complete production before PM service
- PM pastry is mainly execution and plating, not bulk production

### 6. Floater Logic
- CDC can be used as floater
- Raimi can be used as floater
- Natalia may be used as emergency PM support, but should not be the default plan

### 7. Training Awareness
- Training status for a position means that someone who can do this has to do this position alongside the trainee; the trainee counts as 0 person working on the same position. (example: if Tony is training on stuzz and Amy is the only person that day who can do stuzz, then Amy must to stuzz, while Tony trains on Stuzz; together, they count as 1 person because one of them is training on Stuzz).
- When one is training for a position, one must be paired with someone else who can actually do this position.
- If a staff member has both "Can do" and "Training on" capabilities, when they work they must do their "Can do" position AND simultaneously train on the "Training on" position. They count as 1 person for the "Can do" position and 0 for the "Training on" position. (example: if Sam can do Pasta and is training on Sauté, then on a day Sam works, Sam does Pasta [counts as 1] and trains on Sauté [counts as 0]; a separate Sauté-qualified person must also be on Sauté that day).
- If a staff member has only "Training on" capabilities, when they work they only do the training. They count as 0 for that position and must be paired with someone qualified.

### 8. Availability
- Prioritize to use chef, CDC and 2 sous chefs first. 
- For all other staff, especially PM staff, try to maintain a fair schedule (see #10 below).

### 9. Default Stable Interpretation
Under normal conditions, the kitchen should be interpreted as:
- Chef = primary expeditor
- CDC = expediter or support/Sauté/floater
- Raimi = strongest all-around PM flex, preferred on the hot side
- Brandon = default Sauté
- Mateo = default Pantry
- AJ = default PM Pastry
- Sebastian = best cold-side flexible support
- Echo = default stuzz
- Natalia = AM lead with emergency PM support value

### 10. Fairness
- AM staff should be working comparable number of days and resting comparable number of days. 
- PM staff should also be working comparable number of days and resting comparable number of days.
